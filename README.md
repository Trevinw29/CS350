1. Summarize the project and what problem it was solving.
In this project I highlighted my milestone and final project. I did this because it highlights my ability to provide functions to a system in order to get it to work in my desired way. The milestone.py file established core functionalities, such as managing course or customer data, handling state changes, and implementing basic input/output operations. The Thermostat.py project expanded on this by creating a fully operational state machine to manage temperature states (heat, cool, off), interact with hardware peripherals like LEDs, buttons, and a 16x2 LCD display, and simulate sending temperature data to a server over UART.

2. What did you do particularly well?
I successfully implemented the state machine logic for the thermostat, ensuring smooth transitions between heating, cooling, and off states.

3. Where could you improve?
I could improve the code by adding better error handling and validation for user input, especially for button presses and temperature readings. Additionally, the UI for the LCD display could be made more user-friendly

4. What tools and/or resources are you adding to your support network?
I used multithreading with Python's threading library to manage display updates without blocking other operations. These tools expanded my support network for developing hardware-interfacing projects in Python.

5. What skills from this project will be particularly transferable to other projects and/or course work?
This project strengthened my skills in state machine design, hardware-software integration, threading, I2C and UART communication, and peripheral management.

6. How did you make this project maintainable, readable, and adaptable?
It def isnt perfect but the fundamental guidelines were my north star. I made the project maintainable by organizing the code into classes and making minimal edits.
